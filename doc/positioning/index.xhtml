Twisted Positioning Documentation

Introduction

Twisted Positioning is a framework for doing geospatial positioning (trying to find where you are on Earth) using Twisted.

High-level overview

In Twisted Positioning, you write an IPositioningReceiver implementation that will get called whenever some information about your position is known (such as position, altitude, heading...). Twisted Positioning provides a base class called BasePositioningReceiver you might want to use that implements all of the receiver methods as stubs.

Secondly, you will want a positioning source, which will call your IPositioningReceiver. Currently, Twisted Positioning provides an NMEA implementation, which is a standard protocol spoken by many positioning devices, usually over a serial port.

Examples

  • nmealogger.py - Connects to an NMEA device on a serial port, and reports whenever it receives a position.
